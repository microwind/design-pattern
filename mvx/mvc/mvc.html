什么是 MVC？

MVC（Model-View-Controller）是一种经典的软件架构模式，广泛应用于前后端开发中。 它将应用分为三个部分：模型 (Model)、视图 (View) 和控制器 (Controller)，通过清晰的职责分离， 提高代码的可维护性、可扩展性和可测试性。

MVC 组成部分

1. Model（模型）

  • 负责数据的获取、存储和业务逻辑。
  • 通常与后端 API 或数据库交互，提供应用所需的数据。
  • 在前端框架中，Model 可能是数据对象或状态管理工具 (如 Vuex、Redux)。

2. View（视图）

  • 负责用户界面的展示。
  • 通常由 HTML + CSS 组成，提供直观的页面布局和样式。
  • 视图仅负责渲染数据，不直接处理业务逻辑。

3. Controller（控制器）

  • 作为 View 和 Model 之间的桥梁。
  • 接收用户输入，调用 Model 更新数据，并刷新 View。
  • 控制器处理业务逻辑，确保数据流动的正确性。

MVC 的特点

  • 关注点分离：UI 逻辑、业务逻辑和数据逻辑独立，使得代码结构更加清晰。
  • 可维护性强：由于各个组件的职责单一，修改或扩展功能时影响较小。
  • 增强可测试性：业务逻辑集中在控制器和模型中，可以轻松进行单元测试。
